01PAIN POINTS — 預算管理的痛

Excel 管不動、SaaS 套不準的時候

預算這件事跨年、跨單位、跨類別、跨人，Excel 撐不住、套裝 SaaS 又貼不上你獨特的組織結構。最後變成「預算規劃靠 Excel、執行靠記憶、檢核靠補單」。

03ARCHITECTURE — 系統架構

前後端分離，內網一台主機跑得起來

基於開源主流框架，無雲端依賴。所有資料、後端服務、前端 SPA、資料庫均部署在貴司內網一台主機，搭配 IIS 反代提供使用者瀏覽器存取。

07FAQ — 常見問題

常見問題

跟現成 SaaS 預算系統有什麼不同？

SaaS 是「你配合軟體」 — 組織命名、流程階段、報表欄位都得遷就軟體既定樣式。我們是「軟體配合你」 — 從組織單位到簽核流程都依貴司現況設計。代價是建置時間較長（4-8 週 vs SaaS 開帳號就能用）但長期省去「人配合系統」的隱形成本。

系統會放在我們公司內網嗎？資料安全如何？

是的，整套部署在貴司內網一台主機（或 VM）。前端 SPA、後端 API、MySQL 資料庫全部本機執行，無雲端依賴。日常運作不需要對外網路，瀏覽器透過內網或 VPN 存取。資料庫備份、權限管理由貴司 IT 掌握。

可以接我們既有的 HR / ERP 系統嗎？

可以。屬於企業全客製範圍。常見介接：HR 系統同步員工資料、ERP 拉預算與實際支出、採購系統推送決標結果。介接方式以 REST API 或 SQL view 為主，視貴司既有系統而定。

建置完成後，未來我們想自己加功能可以嗎？

可以。我們交付完整原始碼（前端 + 後端 + DB schema + 部署腳本），不綁授權、不藏關鍵程式。貴司若有 Node.js / React 工程師可自行擴充。我們也可在維護合約內持續開發。

多少人可以同時使用？需要按帳號數加價嗎？

不需要。系統部署在貴司內網一台主機，使用人數無限制（受限於該主機效能）。實測一台中階伺服器可支撐 50-100 位使用者同時存取無壓力。

需要哪種伺服器規格？

Windows Server 2019 / 2022（或 Windows 10/11 Pro 也可）、4 核心 CPU、8GB RAM、100GB 硬碟即可起步。Ubuntu / RedHat 也支援，但需另議部署腳本（既有為 Windows 一鍵包）。

導入週期通常多久？

標準版約 4-6 週、進階版 6-10 週、全客製 8-16 週。以「需求訪談 → 客製開發 → 內測 → 教育訓練 → 上線」五階段切，每階段都有里程碑驗收。

付款方式？

分三期：簽約 30%（啟動）/ 內測完成 40%（功能交付）/ 上線驗收 30%。可開立發票、可接公司對公司付款。簽約後到上線約 1.5-3 個月。
